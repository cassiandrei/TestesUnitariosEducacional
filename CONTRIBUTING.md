# Contribuindo para API de Pedidos

Obrigado por considerar contribuir para este projeto educacional! 

## Como Contribuir

### Reportando Bugs
- Use as GitHub Issues para reportar bugs
- Descreva claramente o problema e como reproduzi-lo
- Inclua informações sobre o ambiente (.NET version, OS, etc.)

### Sugerindo Melhorias
- Use as GitHub Issues para sugerir novas funcionalidades
- Explique claramente o valor educacional da sugestão
- Considere se a mudança se alinha com os objetivos didáticos do projeto

### Pull Requests

1. Fork o projeto
2. Crie uma branch para sua feature (`git checkout -b feature/AmazingFeature`)
3. Commit suas mudanças (`git commit -m 'Add some AmazingFeature'`)
4. Push para a branch (`git push origin feature/AmazingFeature`)
5. Abra um Pull Request

### Padrões de Código

- Siga as convenções de nomenclatura do C#
- Mantenha a cobertura de testes alta
- Documente novos conceitos de teste no arquivo `docs/FundamentosTestes.md`
- Use comentários em português para manter consistência

### Testes

- Todos os novos códigos devem ter testes unitários
- Siga o padrão AAA (Arrange, Act, Assert)
- Use nomes descritivos para os testes
- Mantenha os testes simples e focados

## Objetivos do Projeto

Este é um projeto **educacional** focado em demonstrar:
- Fundamentos de testes unitários
- Boas práticas com XUnit e Moq
- Arquitetura limpa em .NET
- Documentação clara e didática

Contribuições que não se alinhem com estes objetivos podem não ser aceitas.

## Código de Conduta

- Seja respeitoso e profissional
- Foque no valor educacional
- Ajude outros a aprender
- Mantenha discussões construtivas

## Dúvidas?

Abra uma Issue com a tag "question" se tiver dúvidas sobre como contribuir.
